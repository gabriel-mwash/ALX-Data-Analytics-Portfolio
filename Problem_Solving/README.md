# Problem-Focused Analysis Project

## Overview

This project explores how to define a problem, craft a concise and clear problem statement, tsk the right questions to determine the clarity and conciseness of a problem statement and what are best approaches or ways to apply to address the problem. 

The goal is to demonstrate how complex problems can be examined systematically before attempting to solve them.

---

## Project Purpose

The purpose of this project is to demonstrate an understanding of:

* What constitutes a **problem** in a real-world context
* How to **formulate a clear and concise problem statement**
* How to ask structured questions to clarify a problem
* How different frameworks can be used to understand and approach a problem

This project therefore focuses on **problem exploration and analysis** rather than immediate implementation of solutions.

## Key Questions

To clarify the problem, the project uses structured questioning methods such as the **5W2H framework**.

The analysis seeks to answer questions such as:

* What exactly is the problem?
* Why is the problem important?
* Who is affected by the problem?
* Where does the problem occur?
* When does the problem occur?
* How does the problem manifest?
* How significant is the impact of the problem?

---

## Analytical Approaches Used

To better understand the problem, the project applies the following approaches:

### 1. Design Thinking

A human-centered approach used to explore the problem from the perspective of those affected.
This includes stages such as empathizing with stakeholders, defining the problem, and exploring potential ideas.

### 2. Pragmatic Thinking

A logical and structured approach used to break down the problem into manageable components.
This may involve conceptual models such as data flow representation and step-by-step logical reasoning.
